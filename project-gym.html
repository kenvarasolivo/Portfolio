01
Architecture & Backend

FLUTTER CLEAN
ARCHITECTURE

Built a modular Flutter application following Clean Architecture principles — strict separation into Presentation, Domain, and Data layers. The backend is powered by Supabase, handling authentication, relational database queries, and cloud media storage. This structure keeps business logic independent from UI and backend implementation, making every layer independently testable and replaceable.

02
Role Management

ROLE-BASED
ACCESS CONTROL

Implemented a role-based permissions system that enforces access at the application level. Verified Users unlock full CRUD functionality, admin controls, and content management. Guests can browse the exercise library but are restricted from any write operations. Roles are stored and validated via Supabase Row Level Security, ensuring the data layer enforces the same rules as the UI.

UI/UX & Features

DARK MODE UI &
INTERACTIVE FEATURES

Designed a responsive Dark Mode interface centred around an interactive muscle map — users tap muscle groups to filter relevant exercises instantly.
