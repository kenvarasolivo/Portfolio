01
Modularisierung & Clean Code

FEATURE-BASED
MODULE ARCHITECTURE

The existing frontend was a monolith — components entangled with business logic, API calls scattered across views. I refactored the entire codebase into encapsulated, feature-based modules: Auth, Conversations, Prompt, and Documents. Business and API logic was extracted into reusable Vue 3 Composables, stripping the views down to pure presentation — each module self-contained, independently testable, and trivial to extend.

02
REST-Ausrichtung & Separation of Concerns

CLEAN SPLIT:
WEB vs. API LAYER

I enforced a strict separation between view delivery and data logic. Route files were split clearly: web.php handles only Inertia.js page rendering, while api.php owns all resource-based endpoints and AI interactions. Controllers became lean — no fat models, no logic leakage. The frontend connects exclusively through resource-based REST endpoints, making the data layer independently replaceable.

03
UI/UX & Streaming-Feedback

AI STREAMING
THINKING SKELETON

AI responses feel instant or broken — there's no middle ground. I redesigned the chat interface to communicate state at every step of the streaming process. While the model generates, users see Thinking Skeletons — animated placeholder lines that pulse in real time — paired with dynamic micro-status indicators that reflect the actual pipeline stage. The result: perceived latency drops to near zero, and users never feel like the app has stalled.
